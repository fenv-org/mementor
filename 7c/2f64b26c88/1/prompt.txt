- Parse each JSON line: check "type" field
  - "user": message.content is a String → wrap in ContentBlock::Text
  - "assistant": message.content is an array of content blocks
    - {"type":"text","text":"..."} → ContentBlock::Text
    - {"type":"thinking","thinking":"..."} → ContentBlock::Thinking
    - {"type":"tool_use","name":"...","input":{}} → ContentBlock::ToolUse
    - {"type":"tool_result","tool_use_id":"...","content":"..."} → ContentBlock::ToolResult
  - "file-history-snapshot": extract file paths from snapshot.trackedFileBackups keys
  - "progress": TranscriptEntry::Progress with the line content
  - "pr-link": TranscriptEntry::PrLink (extract pr_number, pr_url, repository from message)
  - unknown type: TranscriptEntry::Other
- group_into_segments: group entries into ConversationSegment. Each segment starts with a User message and includes all entries until the next User message.
- Use types from `crate::model::*`
- Include comprehensive unit tests with realistic JSONL fixture data

  4. Deserialize into CheckpointMeta
  5. For each session, populate `blob_path` field
- `pub async fn load_checkpoint(checkpoint_id: &str) -> Result<CheckpointMeta>` — derive shard from first 2 chars of ID
